## Short Description

Open Local Digital Twin is an Apache-2.0, self-hosted runtime for building
public-data city baselines, preserving source evidence, separating public,
provider, inferred, and authority-grade layers, and exposing interoperable city
intelligence through open standards.

## Longer Description

Open Local Digital Twin helps cities, universities, civic labs, and
public-interest technology teams build local digital twins from public data
without requiring personal microdata or proprietary provider datasets. The
runtime registers source evidence, consolidates open city inventories, generates
viewer summaries, and exposes standards-native outputs for DCAT, OGC API
Features, NGSI-LD, and FIWARE-compatible workflows.

The project is designed around a clear public-good boundary: open data is
treated as evidence rather than automatic official truth; inferred outputs are
kept separate from authority-grade decisions; provider/private data can be
attached only through governed layers; and public baselines can be reused
without private Polisplexity deployments.

## SDG Relevance

### SDG 9: Industry, Innovation And Infrastructure

The project provides reusable open digital infrastructure for city-scale spatial
data, interoperability, and civic AI workflows. It lowers the cost of building
local digital twin baselines and makes infrastructure planning data more
auditable and reusable.

### SDG 11: Sustainable Cities And Communities

The runtime supports evidence-based urban planning, resilience analysis, public
service mapping, mobility baselines, environmental layers, and city-scale
decision support through transparent public-data workflows.

### SDG 13: Climate Action

Open Local Digital Twin includes patterns for environmental phenomena, terrain,
hydrology, weather-field, green-blue systems, and climate-relevant urban
analysis. These help cities prepare public baselines for adaptation and
resilience planning.

### SDG 16: Peace, Justice And Strong Institutions

The project supports transparency and accountability by preserving provenance,
separating public evidence from official decisions, and documenting governance
boundaries for public/private data.

### SDG 17: Partnerships For The Goals

The runtime is designed for multi-stakeholder collaboration among cities,
universities, civic labs, private providers, and standards communities through
open APIs and interoperability contracts.

## Platform Independence

The core runtime is self-hosted and based on open or widely adopted components:
Node.js, Express, Next.js, PostgreSQL/PostGIS, Docker Compose, and MapLibre.
Closed provider data, hosted services, private deployments, and proprietary
extensions are optional and are not required to run the public-data baseline.

## Data Import / Export

The solution imports and exports non-PII city data through non-proprietary
formats and APIs, including GeoJSON, CSV-compatible workflows, JSON, JSON-LD,
Mapbox Vector Tiles, DCAT, OGC API Features, NGSI-LD entity projections, and
FIWARE-compatible context workflows.

## Privacy And Applicable Laws

The public baseline does not require personal microdata. Deployments adding
private or personal data must configure lawful access, retention, access
control, publication review, and jurisdiction-specific compliance.

Evidence:

https://github.com/polisplexity/open-local-digital-twin/blob/main/PRIVACY.md

## Standards And Best Practices

The runtime aligns with DCAT, OGC API Features, NGSI-LD, FIWARE-compatible
context workflows, PostGIS-backed geospatial storage, source provenance,
license/attribution metadata, bounded payloads, and vector tiles for city-scale
rendering.

## Do No Harm By Design

The project separates public evidence, inferred outputs, provider layers, and
authority-grade decisions. It avoids treating open data or AI inference as
official truth without human and institutional review.

Evidence:

https://github.com/polisplexity/open-local-digital-twin/blob/main/DO_NO_HARM.md
